# 新闻时效性改进方案

## 问题分析

### 原有问题
1. **日期解析不准确**：所有新闻都被标记为抓取当天的日期，而非实际发布日期
2. **没有过期清理**：旧新闻会一直保留，导致"最新新闻"中包含过期内容
3. **消息源有限**：只有机器之心和量子位两个来源

### 改进措施

## V2 版本改进

### 1. 正确解析发布日期
- 支持相对日期解析："今天", "昨天", "X天前", "X小时前"
- 支持标准格式："MM月DD日", "YYYY-MM-DD"
- 自动处理跨年情况

### 2. 时效性验证
- 只保留最近 **7天** 内的新闻
- 自动清理超过7天的新闻文件
- 在控制台输出时效性统计

### 3. 扩展消息源
- 机器之心 (jiqizhixin.com)
- 量子位 (qbitai.com)
- TechCrunch AI
- RSS 订阅源 (AI科技评论、InfoQ等)

### 4. 自动清理机制
- 每次更新时自动删除过期新闻文件
- 保留最多200条新闻在data/news.json中
- 只为前10条新新闻创建内容文件

## 配置参数

```python
# 新闻有效期（天）
MAX_NEWS_AGE_DAYS = 7

# 最大保留新闻数
MAX_NEWS_COUNT = 200

# 每次创建内容文件的数量
MAX_CONTENT_FILES = 10
```

## 文件变更

### 新增文件
- `scripts/scraper/news_scraper_v2.py` - 改进版新闻爬虫

### 修改文件
- `scripts/scraper/scraper.py` - 集成V2版本和清理功能

## 使用方式

自动更新脚本已集成时效性检查，正常运行即可：

```bash
./auto-update.sh
```

## 预期效果

1. 新闻日期显示为实际发布日期，而非抓取日期
2. 只展示最近7天的新闻，确保时效性
3. 过期新闻自动清理，保持内容新鲜
4. 消息源更丰富，覆盖面更广
